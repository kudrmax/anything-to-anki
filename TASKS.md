## Важно и срочно

- [важность: high, срочность: med] Выбор лучшей фразы для target'а (quick win, фундамент уже есть)

  Самый дешёвый и самый ценный пункт во всём списке. Метрика `fragment_unknown_count` уже считается, но pipeline берёт первую найденную фразу, а не минимум по unknown_count. То есть инфраструктура для «target — единственный незнакомый элемент» уже есть, но не используется для отбора.

  Нужно: при дедупликации кандидатов по `(lemma, pos)` оставлять не первое вхождение, а фразу с минимальным `fragment_unknown_count` (тай-брейкер — длина в sweet spot). #tech #product

- [важность: med, срочность: high] Унифицировать и улучшить UI редактирования границ слова и создания нового слова

  Сейчас два сценария — редактирование границ существующего target'а и создание нового слова вручную — не унифицированы и не очень удобны.

  Привести их к единому, удобному UX (общий компонент, общие действия, понятная навигация). #tech #product

- [важность: high, срочность: high] Кастомный промпт для слова + набор заготовленных промптов

  Добавить возможность для слова вбивать кастомный любой промпт. Это полезно для сложных слов, где ты НЕ понял значение и тебе нужно более конкретное объяснение.

  Также — набор заготовленных промптов типа «не понял, объясни подробнее». #tech #product

- [важность: high, срочность: low] Улучшить выбор границ фраз при сегментации

  Сегментация — один из ключевых шагов pipeline (источник → текст → фразы → target'ы → карточки), и качество карточек напрямую зависит от того, насколько «логически законченными» получаются фразы.

  Цель — улучшить алгоритм выделения границ так, чтобы фраза вокруг target'а действительно была самодостаточной единицей контекста. #tech #product

- [важность: high, срочность: low] Улучшить отбор target'ов — чтобы вверху было реально самое полезное

  Шаг «Отбор target'ов» из pipeline: приоритизация по частотности, CEFR, фразовым глаголам, исключение известных пользователю слов, плюс приоритет фраз, где всё кроме target'а уже знакомо.

  Целевое состояние: вверху списка оказывается реально самое полезное для пользователя. Нужно посмотреть, что мешает этому сейчас, и улучшить ранжирование. #tech #product

- [важность: high, срочность: low] YouTube как источник

  `SourceType` поддерживает только TEXT/LYRICS/SUBTITLES/VIDEO (локальные файлы). YouTube — самый частый сценарий для learners, и его нет вообще.

  yt-dlp ставит и субтитры, и аудио, и метаданные. Это расширяет аудиторию сильнее всего остального. #tech #product

- [важность: high, срочность: low] Детектор коллокаций (phrasal verbs закрыты, коллокации — нет)

  Архитектурно лежит ровно туда же, куда `phrasal_verb_detector`. #tech #product
