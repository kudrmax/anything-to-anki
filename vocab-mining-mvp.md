# VocabMiner — MVP-спецификация

*Приложение для изучения английских слов из контента, который ты потребляешь*

---

## Суть продукта

Ты закидываешь любой контент (песня, видео, текст) → система извлекает текст →
подсвечивает слова, которые тебе стоит выучить → ты подтверждаешь или отклоняешь →
из подтверждённых генерируются карточки Anki.

---

## Архитектура: пайплайн обработки

### Слой 1 — Извлечение текста

Конвертация любого источника в сырой текст.

- **YouTube** → transcript API (в MVP: ручная вставка)
- **Музыка** → Genius API (в MVP: ручная вставка)
- **Plain text / отрывок** → принимается как есть
- **Книга** → парсинг формата (в MVP: ручная вставка)

**Вход:** URL или raw content
**Выход:** сырой текст + метаданные источника (название, автор, тип)

### Слой 2 — Очистка текста

Текст приводится в чистый, читаемый вид. Грамматика, пунктуация и структура
сохраняются полностью — текст должен оставаться пригодным для цитирования
в карточках Anki.

**Что убираем:**
- Лишние пробелы, дублирующиеся переносы строк
- Технический мусор: таймкоды субтитров (`[00:01:23]`), теги разметки,
  служебные метки (`[Music]`, `[Applause]`, `♪`)
- Дублирующиеся строки (частая проблема в субтитрах и lyrics)
- BOM-маркеры, невидимые unicode-символы

**Что НЕ трогаем:**
- Знаки препинания (запятые, точки, апострофы, тире)
- Междометия (oh, well, hmm — это часть языка)
- Сленг и сокращения (gonna, wanna — могут быть объектами изучения)
- Заглавные буквы, кавычки, восклицательные/вопросительные знаки
- Структуру строф (для lyrics) или абзацев (для прозы)

**Вход:** сырой текст
**Выход:** чистый текст, сохранивший исходную грамматику и структуру

### Слой 3 — Анализ и отбор слов

Большой аналитический слой, который принимает чистый текст и выдаёт
ранжированный список слов-кандидатов с контекстными фрагментами.

#### 3.1 — Токенизация и лемматизация (spaCy)

- Разбиение текста на предложения и токены
- Лемматизация каждого токена (running → run, better → good)
- Определение части речи (POS)
- Построение dependency tree (нужно для извлечения фрагментов на шаге 3.5)
- Автоматическое исключение нерелевантных токенов:
  имена собственные, числа, знаки препинания

#### 3.2 — Классификация уровня (cefrpy)

Каждая уникальная лемма получает CEFR-уровень через `cefrpy`:
A1 = 1.0, A2 = 2.0, B1 = 3.0, B2 = 4.0, C1 = 5.0, C2 = 6.0.
Слова, отсутствующие в базе cefrpy, помечаются как «неизвестный уровень»
и трактуются как сложные (C1+).

#### 3.3 — Фильтрация по профилю пользователя

Пользователь при онбординге указывает свой уровень (например, B1).

**Правило отбора кандидатов:** показываем слова с уровнем > уровень пользователя.
Для B1-пользователя: показываем B2, C1, C2.

**Whitelist известных слов:** слова, которые пользователь ранее отметил как «знаю»,
исключаются. Whitelist растёт с каждым обработанным текстом.

**Blacklist отклонённых слов:** слова, отмеченные как «не интересно»,
не предлагаются повторно (или предлагаются через настраиваемый интервал,
например 3 месяца).

#### 3.4 — Ранжирование кандидатов

Отфильтрованные слова сортируются по полезности.

Факторы ранжирования (детерминированные, без AI):

1. **Частотный бэнд (BNC/COCA)** — слова из диапазона K-3000–9000 получают
   приоритет (самая «окупаемая» зона для L2-учащихся)
2. **Количество появлений в источниках пользователя** — слово, встреченное
   в 3 песнях, приоритетнее слова из одной
3. **Фразовые глаголы и коллокации** — приоритетнее одиночных слов
   (get away > ubiquitous), определяются через spaCy dependency parsing

#### 3.5 — Извлечение контекстных фрагментов

Для каждого слова-кандидата извлекается минимальный осмысленный фрагмент
оригинального текста — не полное предложение и не отдельное слово,
а фраза из 3–12 слов, дающая достаточно контекста для понимания.

**Алгоритм (на основе spaCy dependency tree):**

1. Начинаем с целевого слова
2. Берём subtree текущего узла (`token.left_edge` → `token.right_edge`)
3. Если subtree < 3 слов — поднимаемся к `token.head` и повторяем
4. Если subtree попал в коридор 3–12 слов — это наш фрагмент
5. Если subtree > 12 слов — берём окно ±5 слов вокруг целевого слова,
   обрезанное по границе предложения, с многоточием на краях

Алгоритм проверен на реальном тексте — spaCy гарантирует, что subtree
для английских моделей всегда contiguous (токены идут подряд),
что позволяет безопасно вырезать span.

**Оценка чистоты фрагмента:**
Подсчитываем, сколько других незнакомых слов (выше уровня пользователя
И не в whitelist) содержится во фрагменте.
- 0–1 → фрагмент чистый ✅
- 2+ → фрагмент грязный ⚠️ (показываем предупреждение)

Примеры извлечённых фрагментов (проверено через spaCy):

| Слово | Фрагмент | Слов | Метод |
|-------|----------|------|-------|
| relentless | The **relentless** pursuit of perfection | 5 | subtree head |
| overwhelming | the **overwhelming** urge to overcommit | 5 | subtree head |
| sacrifice | **sacrificing** their well-being in the process | 6 | own subtree |
| pretend | **pretending** they don't exist | 4 | own subtree |
| burnout | ...often leads to **burnout** | 4 | window |

## Генерация карточек Anki

### Лицевая сторона (вопрос)

Контекстный фрагмент из слоя 3.5, целевое слово выделено жирным:

> ...***sacrificing*** their well-being in the process

Внизу мелким шрифтом — источник: «текст: "Статья о продуктивности"»

### Оборотная сторона (ответ)

**MVP-реализация (словарный API, без AI):**

Данные берутся из Free Dictionary API (https://dictionaryapi.dev/):

- **Слово** в базовой форме (лемма)
- **Транскрипция** (IPA)
- **Определение** на английском (первое значение, совпадающее по POS)
- **Перевод** на русский (если доступен в API; иначе — пропускаем)
- **Часть речи**
- **Пример использования** (из словаря, если есть)

Пример:
```
sacrifice  /ˈsækrɪfaɪs/
verb

an act of giving up something valued for the sake
of something else regarded as more important

Перевод: жертвовать, приносить в жертву

Example: "we had to sacrifice quality for speed"
```

**Известная проблема:** у полисемичных слов (run, set, light) словарь
вернёт несколько значений, и непонятно, какое соответствует контексту.
В MVP берём первое значение, совпадающее по части речи.

**Архитектура под будущее AI-обогащение:**

Генерация оборотной стороны вынесена в отдельный интерфейс `CardBackGenerator`:

```python
class CardBackInput:
    lemma: str              # базовая форма слова
    pos: str                # часть речи
    original_fragment: str  # контекстный фрагмент из источника
    source_title: str       # название источника
    source_type: str        # тип (lyrics, video, text...)
    user_level: str         # CEFR-уровень пользователя

class CardBackOutput:
    word: str               # слово в базовой форме
    transcription: str      # IPA
    definition: str         # определение на английском
    translation: str        # перевод на русский
    pos: str                # часть речи
    example: str | None     # пример из словаря
    context_note: str | None  # [AI] пояснение к контексту
    image_url: str | None     # [AI] картинка
    mnemonic: str | None      # [AI] мнемоническая подсказка
```

MVP-реализация `DictionaryCardBackGenerator` заполняет только базовые поля.
AI-поля (`context_note`, `image_url`, `mnemonic`) всегда None.

Будущая AI-реализация `AICardBackGenerator` с тем же интерфейсом сможет:
- Выбирать правильное значение слова по контексту фрагмента
- Генерировать определение на уровне пользователя
- Добавлять мнемонические подсказки
- Подбирать или генерировать картинку
- Переводить с учётом контекста

Подмена реализации — одна строка в конфиге, без изменения остального кода.

## Пользовательский флоу (MVP)

Приложение запускается локально на ноутбуке (Docker Compose или два терминала).
Открывается в браузере на localhost. Четыре экрана.

### Экран 1 — Inbox (Входящие)

Главный экран. Список источников + поле добавления нового.

- Поле ввода: textarea для текста + название + тип (text / lyrics / subtitles)
- Кнопка «Добавить» → источник сохраняется со статусом «new»
- Можно накидать несколько источников подряд и закрыть браузер
- Кнопка «Обработать» на каждом источнике → запуск пайплайна (слои 1–3)
- Кнопка «Обработать все» → батчевая обработка всех новых
- После обработки: показываем количество кандидатов, кнопка «Разметить →»
- Для частично размеченных: «Продолжить разметку», счётчик прогресса
- Для размеченных с отметками «учить»: кнопка «Экспорт»

### Экран 2 — Разметка (/sources/{id}/review)

Центральный экран. Две панели.

**Левая панель:** оригинальный текст, слова-кандидаты подсвечены цветом
по уровню (B2 — жёлтый, C1 — оранжевый, C2 — красный).
Клик по слову → скролл к карточке справа.

**Правая панель:** список кандидатов, отсортированный по полезности.
Каждая карточка: слово, CEFR, бэнд, контекстный фрагмент, индикатор чистоты.
Три кнопки: «Учить» / «Знаю» / «Не интересно».
При наведении на карточку → подсветка слова в тексте слева.

Счётчик внизу: «Размечено: 5/12 · К изучению: 3»
Кнопка «Сохранить и выйти» → сохранение прогресса, возврат в Inbox.
Неразмеченные слова сохраняют статус null — можно вернуться позже.

### Экран 3 — Экспорт (/sources/{id}/export)

Превью карточек перед генерацией .apkg.

- Список карточек с превью лицевой и оборотной стороны
- Кнопка «Скачать .apkg» → скачивание файла для импорта в Anki

### Экран 4 — Настройки (/settings)

- Выбор CEFR-уровня пользователя
- Просмотр / редактирование whitelist (известные слова)
- Просмотр / редактирование blacklist (не интересно)
- Интервал blacklist (через сколько месяцев предлагать снова)

## Модель данных (MVP)

### User
```
- id
- display_name
- cefr_level: enum (A1, A2, B1, B2, C1, C2)
- created_at
```

### KnownWord
```
- user_id
- lemma: str
- pos: str
- status: enum          # "known" | "not_interested"
- marked_at: datetime
- expires_at: datetime | null  # для not_interested: когда снова предлагать
```

### Source
```
- id
- user_id
- type: enum            # "text" | "lyrics" | "subtitles"
- title: str
- original_text: text   # сырой текст (слой 1)
- cleaned_text: text    # очищенный текст (слой 2)
- status: enum          # "new" | "processed" | "partially_reviewed"
                        #   | "reviewed" | "exported"
- created_at
```

### WordCandidate
```
- id
- source_id
- lemma: str
- pos: str
- cefr_level: float
- frequency_band: int          # K-позиция в BNC/COCA
- context_fragment: str         # фрагмент из слоя 3.5
- fragment_unknown_count: int   # кол-во незнакомых слов во фрагменте
- user_action: enum | null      # "learn" | "known" | "skip"
- acted_at: datetime | null
```

### AnkiCard
```
- id
- user_id
- word_candidate_id
- front_html: text
- back_html: text
- back_data: json              # структурированные данные (CardBackOutput)
                               # храним отдельно для будущей перегенерации через AI
- exported: bool
- exported_at: datetime | null
```

## Что НЕ входит в MVP

- Автоматическое извлечение текста из URL (YouTube, Genius) — ручная вставка
- AI-обогащение карточек (выбор значения по контексту, мнемоники, картинки)
- AI-предпочтения по тематике (бизнес / литература / разговорный)
- Работа с книгами / длинными текстами (ограничение: ~2000 слов)
- Интеграция с Anki через AnkiConnect (только экспорт файла)
- Мобильное приложение (web first)
- Мультиязычность (только English → Russian)
- Трекинг прогресса / аналитика
- Spaced repetition внутри приложения (делегируется Anki)

---

## Метрики успеха MVP

1. **Точность фильтрации:** ≥80% предложенных слов получают «учить»
   (а не «знаю» или «не интересно»)
2. **Качество фрагментов:** ≥80% фрагментов чистые (0–1 лишнее слово)
3. **Конверсия:** ≥70% обработанных источников → хотя бы одна карточка
4. **Рост whitelist:** уменьшение ложных кандидатов с каждым источником
