# TTS: тестирование провайдеров (фаза 1)

## Проблема

Для TEXT и LYRICS источников аудио предложения нет — карточка Anki остаётся без аудио-компонента. Нужен TTS-провайдер для генерации озвучки. Перед реализацией фичи — тестируем 5 бесплатных провайдеров и выбираем лучший на слух.

## Провайдеры (от ожидаемо лучшего к менее хорошему)

1. **edge-tts** — нейронные голоса Azure, бесплатно, без API-ключа, нужен интернет. Пакет: `edge-tts`
2. **Kokoro** (82M) — офлайн, ~350MB модель, студийное качество. Пакет: `kokoro`, `soundfile`
3. **Coqui XTTS v2** — офлайн, ~2GB, voice cloning. Пакет: `coqui-tts`
4. **Piper** — офлайн, ~75MB, быстрый. Пакет: `piper-tts`
5. **gTTS** — Google Translate TTS, бесплатно, без ключа, нужен интернет. Пакет: `gTTS`

## Что делает скрипт

- Принимает текст предложения (default: `"She couldn't help but procrastinate, even though the deadline was looming."`)
- Для каждого провайдера генерирует аудиофайл в `data/tts_test/`
- Имена файлов: `01_edge_tts.mp3`, `02_kokoro.mp3`, `03_coqui.mp3`, `04_piper.mp3`, `05_gtts.mp3`
- Выводит в консоль результат: успех/ошибка для каждого провайдера, пути к файлам
- Если провайдер не установлен — пропускает с сообщением

## Результат

Пользователь слушает файлы, выбирает провайдера. После этого — реализация TTS-фичи по спеку `2026-04-24-tts-sentence-audio-design.md`.
