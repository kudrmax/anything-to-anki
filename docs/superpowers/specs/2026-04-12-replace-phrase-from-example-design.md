# Replace Phrase from Example — Design Spec

## Проблема

Пользователь видит карточку с интересным target-словом, но фраза (`context_fragment`) неудобная: слишком сложная, содержит другие незнакомые слова, плохо демонстрирует контекст. Хочется быстро создать новую карточку с тем же target, но с лучшей фразой.

## Решение

Кнопка в карточке кандидата, которая позволяет выбрать одну из уже сгенерированных AI-примеров (`examples` из meaning) и создать новую карточку. Старая карточка автоматически помечается как `skip`.

---

## UX-флоу

1. Пользователь смотрит на карточку кандидата, у которого сгенерирован meaning с examples.
2. В карточке, рядом с кнопкой редактирования границ, появляется кнопка "Replace phrase" (или иконка).
3. Клик → popover в стиле AI follow-up действий со списком распарсенных примеров.
4. Если примеры не распарсились или их нет — плашка "Примеры недоступны".
5. Клик по примеру → одним действием:
   - Старая карточка получает `status = skip`
   - Создаётся новая карточка с выбранным примером как `context_fragment`
   - Новая карточка ставится в очередь на генерацию meaning
6. UI обновляется: старая карточка уходит в reviewed (skip), новая появляется в pending.

### Кнопка доступна только если

- У кандидата есть meaning
- `examples` не пустой
- Примеры успешно парсятся (split по `\n`, фильтр пустых строк)

## Модель данных

### Таблица `candidates` — новое поле

| Поле | Тип | Default | Описание |
|------|-----|---------|----------|
| `has_custom_context_fragment` | BOOLEAN | FALSE | Фраза не из исходного текста, а выбрана пользователем из AI-примеров |

### Новый кандидат создаётся с полями

| Поле | Значение |
|------|----------|
| `source_id` | Копируется от оригинала |
| `lemma` | Копируется |
| `pos` | Копируется |
| `is_phrasal_verb` | Копируется |
| `surface_form` | `null` (highlight найдёт через lemma-паттерн) |
| `context_fragment` | Текст выбранного примера (без markdown bold) |
| `has_custom_context_fragment` | `true` |
| `cefr_level` | Копируется |
| `zipf_frequency` | Копируется |
| `is_sweet_spot` | Копируется |
| `fragment_purity` | Копируется |
| `occurrences` | `1` |
| `status` | `pending` |

## Backend API

### POST `/candidates/{candidate_id}/replace-with-example`

**Request:**
```json
{
  "example_text": "She was running out of patience."
}
```

**Логика:**
1. Загрузить оригинального кандидата по `candidate_id`
2. Поставить оригиналу `status = skip`
3. Создать нового кандидата (поля из таблицы выше)
4. Поставить нового кандидата в очередь на генерацию meaning
5. Вернуть нового кандидата в ответе

**Ошибки:**
- `404` — кандидат не найден
- `400` — `example_text` пустой

**Response:** `StoredCandidateDTO` (новый кандидат)

## Frontend

### Popover с примерами

- Парсинг: split `examples` по `\n`, фильтр пустых строк, strip markdown bold (`**word**` → `word`)
- Стиль: как у AI follow-up действий (explain grammar и т.д.)
- Плашка "Примеры недоступны" если парсинг дал пустой список

### Обновление UI после замены

- Оригинальная карточка обновляется как `skip` в UI
- Новая карточка появляется в списке pending-кандидатов
- Popover закрывается

---

## Подсветка в тексте справа

- Кандидаты с `has_custom_context_fragment=true` **пропускаются** при поиске fragment в тексте (не вызываем `indexOf` для их `context_fragment`)
- Target (lemma) по-прежнему подсвечивается как mark в тексте — это не меняется

---

## Что НЕ меняется

- **Anki-экспорт** — `highlight_all_forms` подсветит target через lemma-паттерн
- **Генерация meaning** — промпт получит новый `context_fragment` как контекст
- **Сортировка кандидатов** — новый кандидат — обычный pending
- **Существующие эндпоинты** — без изменений
