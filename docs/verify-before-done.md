# Чек-лист перед завершением задачи

Всё — вручную: pre-commit hooks и CI в проекте нет, цена пропущенной проверки — сломанный Docker-билд.

## Backend (если трогал Python-код)

```bash
make lint         # ruff check .
make typecheck    # mypy backend/src (strict)
make test         # pytest (unit + integration)
```

Все три должны пройти без ошибок.

## Frontend (если трогал `frontends/web`)

```bash
cd frontends/web
npm run lint      # eslint
npm run build     # tsc -b && vite build
```

Именно `npm run build`, а **не** `tsc --noEmit` — build полностью повторяет то, что делает Docker при сборке, и ловит ошибки, которые `--noEmit` пропустит.

## Правила

- **Сломавшиеся чужие тесты/линтер — чинить, не скипать.** Сначала основная задача, потом сразу найденные проблемы.
- **Никаких** `--no-verify`, `pytest -k 'not broken'`, `@pytest.mark.skip` на живых тестах, `# noqa` без обоснования в комментарии рядом. Если правило мешает — обсуждаем, не обходим.
- **Если упало что-то не по твоей теме** — сообщить пользователю явно, не прятать и не замалчивать.
- **Если тест flaky** — это тоже баг, чинить, а не запускать повторно до зелёного.
